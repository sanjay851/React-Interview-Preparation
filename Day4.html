Q1.What is UseRef Hook ?(Implementation)

Answer: useRef(initialValue)is a built-in React hook that accepts one argument as the initial value and returns a reference. A reference is an object having a single property “current”, which can be accessed and changed (mutated).

Q2.What is Context api?

Answer: It creates a context object. The React Context API is a component structure, which allows us to share data across all levels of the application. The main aim of Context API is to solve the problem of prop drilling (also called "Threading").

Q3.Difference between callback and useCallback Hook ?

Answer: callback: callback is javascript function,when argument passed to an function whereas another one is hook, in this you pass another function as an argument.

useCallback Hook: The useCallback hook is used Pass an inline callback and an array of dependencies. useCallback will return a memoized version of the callback that only changes if one of the dependencies has changed.

Q4.What is Props Drilling Concept ?What is State Uplifting ?

Answer:
Props Drilling Concept: Prop drilling is basically a situation when the same data is being sent at almost every level due to requirements in the final level.

State Uplifting: by Lifting up the state we make the state of the parent component as a single source of truth and pass the data of the parent in its children.

Time to use Lift up the State: If the data in “parent and children components” or in “cousin components” is Not in Sync.

Example : If we have 2 components in our App. A -> B where, A is parent of B. keeping the same data in both Component A and B might cause inconsistency of data.

Q6.What are React Life cycles Explain each one with Example?

Answer: Each component in React has a lifecycle which you can monitor and manipulate during its three main phases.

  • Mounting
  • Updating
  • Unmounting

Mounting :- It instance of creating or inserting a elements into the DOM.



React has four built-in mounting methods :

1. constructor(): used to initialize an object's state in a class. it is called before the component is mounted.

2. getDerivedStateFromProps(): is used before rendering in the DOM.

3. render(): method is required and will always be called, the others are optional and will be called if you define them.

4. componentDidMount(): allows us to execute the react code when the component is already placed in the DOM.



Updating:- The component is updated whenever there is a change in the component's state & props.

React has five built-in updating methods :

1. getDerivedStateFromProps(): used to update the state, before rendering the component, which depends on props.

2. shouldComponentUpdate(): It only updates the component if the props passed to it changes. and it aviod calling exits react component from re-render

3. render(): when component gets updated, it has to re-render the HTML to the DOM, with the new changes.

4. getSnapshotBeforeUpdate(): method is invoked just before the DOM is being rendered. It is used to store the previous values of the state after the DOM is updated.

5. ComponentdidUpdate(): allows to execute the react code when the component is updated. when the props passed to the component changes are coded here.


Unmounting:- The component is removed from the DOM.

React has one built-in updating methods :

1. ComponentWillUnmount(): allows to execute the react code when the component is destoryed or Unmounting from the DOM. before the component gets unmounted.
