Q1.What is Virtual DOM?

Answer:Virtual DOM is an in-memory representation of real DOM. It is a lightweight JavaScript object which is a copy of Real DOM. in this behid,there are two doms,first one is copy of real dom another is a updating dom , Now when any user makes any changes on the screen like button click because of which the state variable will get updated so in this case the changes will not directly go to Real DOM . Now these two virtual doms will get compared with each other and will check for the new changes. if any changes then virtual dom to be modified ,again virtual dom one sync with the real dom ,and real dom will be getting updated. this complete procedure is known as diffing algorithm.Now the new changes will be updated in your Real dom. this procedure is known as Recoinciliation.

Q3.What is difference between class and functional component?

Answer:
Class Components:

  • class components are stateful components(In React, a stateful component is a component that holds some state like this. setState)
  • Class components uses the react Lifecycle Method
  • class component syntex a little bit complex
  • Constructor structure is required since these are class components.
  • Contains state variable
  • example: constructor(props) { super(props); this.state = {name: ‘ ‘} }
  • It must have the render() method returning JSX (which is syntactically similar to HTML)
Functional Components:
  • Functional Components are stateless components(Stateless components are those components which don't have any state at all, which means you can't use this. setState inside these components.)
  • These components dones not have lifecyle method ,they have hooks like useEffect
  • There is no render method used in functional components.
  • function component syntex is easy
  • No need of constructor
  • Don't have state variable ,they have hooks like useState.
  • There is no render method ,only return method used in functional components.

Q4.What does mean by state and its use in react?

Answer: State of a component is an object that holds some information that may change over the lifetime of the component. We should always try to make our state as simple as possible and minimize the number of stateful components. 

                    class Greeting extends React.Component {
                        constructor() {
                          super(); 
                          this.state = {
                            name: 'John Smith'
                          }
                        }
                        render() {
                          return 
        Hello, my name is { this.state.name }
        ;
                        }
                      }

  • State is similar to props, but it is private and fully controlled by the component ,i.e., it is not accessible to any other component till the owner component decides to pass it.

Q5.What is JSX and why do we use it instead of js?

Answer: JSX stands for JavaScript XML. It is simply a syntax extension of JavaScript. It allows us to directly write HTML in React (within JavaScript code). It is easy to create a template using JSX in React, but it is not a simple template language instead it comes with the full power of JavaScript. Basically it just provides syntactic sugar for the React.createElement() function, giving us expressiveness of JavaScript along with HTML like template syntax. You can embed any JavaScript expression in JSX by wrapping it in curly braces. 

                    const element = React.createElement(
                        'h1',
                        {"className":"greeting"},
                        'hello , World!'
                    )

Q6.What is package.json?

Answer: in package.json files will be having all related files or we can say updated information for files or packages if we installing any packages,in this file we can check that files install or not. .The package. json file is the heart of any Node project.
